# Integration authority policy

## public_phase1_integration_rules

The public Phase 1 projection retains these rules: exactly one registered integrator owns one plan/worktree after a separate maintainer-only `INTEGRATION_AUTHORIZED` decision; the root executor reuses current targeted verification and runs the final repository gate once for the immutable candidate after cold review but before that decision; only approved units and bounded preapproved conflicts may later be integrated; the exact-subject public-safety result uses the required structured audit categories and evidence references; accepted in-scope failures return to root implementation until repaired; and no branch/worktree deletion or cleanup is implied. No final-verifier agent is created. Private plan IDs, local paths, recovery references, and provider identifiers are omitted.
