# Verification-result reuse policy

## public_phase1_rules

The Phase 1 public projection retains only these durable rules: bind results to exact subject identity, command semantics, scope, relevant environment and dependency identity, side effects, and invalidation; query reusable results before running checks; and have reviewers run finding-specific checks only. After cold review when required and final assembly, the root executor runs the full repository gate exactly once for that immutable candidate revision, before the maintainer integration decision. An unchanged candidate reuses that result and receives zero redundant reruns; only recorded evidence loss can justify a replacement run. A content change creates a new identity and invalidates the old candidate result. Numeric packet or context sizes are telemetry only and never trigger workflow action by themselves. Relevant untracked or generated inputs require stable content identities, roles, and scopes or the result is `NON_REUSABLE`.
